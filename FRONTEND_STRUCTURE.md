# Структура фронтенда

```text
legacy-frontend/
├── public/                         # статические файлы Vite
├── src/
│   ├── api/                        # функции HTTP-запросов к backend API
│   │   ├── category.tsx            # категории, карточки, удаление/обновление карточек
│   │   └── user.tsx                # авторизация, пользователи, должности, отделы, история
│   ├── assets/                     # иконки и изображения
│   ├── components/                 # общие UI-компоненты
│   │   ├── MarkdownRenderer.tsx    # Markdown, чекбоксы, таблицы, ссылки, цитаты
│   │   ├── ErrorBanner.tsx         # общий вывод ошибок/успехов
│   │   ├── MultiSelect.tsx
│   │   └── ...
│   ├── hooks/                      # React hooks
│   │   ├── use-permissions.tsx     # проверка прав по должностям
│   │   ├── use-user.tsx
│   │   └── use-category.tsx
│   ├── pages/                      # страницы приложения
│   │   ├── account-list/           # список аккаунтов, деактивация/активация
│   │   ├── card/                   # просмотр/редактирование карточки
│   │   ├── home/                   # темы/категории и карточки
│   │   ├── positions/              # редактирование должностей и прав
│   │   └── user/                   # профиль, история просмотров
│   ├── stores/                     # Zustand-хранилища
│   ├── styles/                     # глобальные стили и markdown.css
│   ├── types/                      # общие TS-типы
│   ├── utils/                      # утилиты, обработка ошибок API
│   ├── App.tsx
│   ├── main.tsx
│   └── routes.tsx
├── package.json
├── tsconfig*.json
└── vite.config.ts
```

## Основные изменения

- Удаление карточки добавлено на странице карточки и доступно только admin/председателю/замам.
- Удаление темы/категории убрано из интерфейса.
- История просмотров завязана на id текущего пользователя в query key.
- Markdown-редактор получил горячие клавиши и предпросмотр справа.
- Чекбоксы Markdown кликабельны и сохраняют текст карточки на backend с debounce.
- Добавлены кнопки деактивации/активации аккаунтов.
- Ошибки API приводятся к нормальной строке вместо `[object Object]`.
